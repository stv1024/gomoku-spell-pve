# 技能五子棋 PVE — 设计文档

## 一、项目目的

验证一个想法：**AI 能否在游戏运行时动态生成游戏机制？**

玩家输入一个四字成语，AI 实时将其解读为一个五子棋技能，生成可执行代码，立即在棋盘上生效。技能不是开发阶段预写的，而是运行时由 AI 现场创造的。

产品形态是一个单机 PVE 闯关五子棋，带轻度 Roguelike 随机性。玩家执白子，面对预置了大量黑子的劣势棋盘，每 3 回合抽取一张成语技能卡，利用技能翻盘连五过关。

这是一个噱头型实验产品，核心体验是"输入成语 → 看到意想不到的技能效果 → 用它翻盘"这个惊喜循环。

## 二、设计思路

### 2.1 为什么是 PVE 而不是 PVP

PVP 需要联网匹配、双方技能交互、信息隐藏、公平性保障，每一项都是重大工程。PVE 砍掉了所有这些：不需要联网，技能单向作用于棋盘，AI 对手不使用技能，不需要考虑对称平衡。开发周期短，也更容易发布给别人玩。

### 2.2 棋盘永远是标准状态

这是最重要的简化决策。技能执行前后，棋盘都是一个合法的标准五子棋棋盘——每个格子要么是黑子、要么是白子、要么是空的。没有 buff，没有隐藏棋子，没有地形效果，没有持续状态。

这带来三个好处：

1. **AI 对手极其简单**。它每回合只需要看一个 15×15 的三值数组，用任何标准的五子棋算法落子即可，不需要理解任何特殊状态。
2. **玩家理解零成本**。所见即所得，不需要学习各种图标和状态含义。
3. **引擎复杂度极低**。不需要事件系统、不需要状态管理、不需要 buff 计时器。

代价是技能表达力受限——不能做"未来 3 回合内对手看不到你的落子"这类时间维度的效果。但对于一个噱头型产品，棋子的增删移变已经足够产生丰富的惊喜感。

### 2.3 技能是瞬时的棋盘编辑

所有技能的本质都是：读取当前棋盘 → 做一组修改 → 结束。五种原子操作的组合覆盖了所有可能的效果：

- **放置**（place）：在空位放一颗棋子
- **移除**（remove）：拿走一颗棋子
- **变色**（transform）：黑变白或白变黑
- **移动**（move）：把棋子从一个位置挪到另一个位置
- **交换**（swap）：两个位置的内容互换

比如"偷梁换柱"就是 transform（把一颗黑子变白），"釜底抽薪"就是 remove（移除关键位置的黑子），"乾坤大挪移"就是多次 swap（交换一个区域内的棋子）。

### 2.4 技能使用不替代落子

玩家每回合可以先使用一张技能卡改变棋盘布局，然后仍然必须正常落一颗白子。技能是"先手优势"而不是"替代行动"。这保证了五子棋本身的策略性不会被技能系统淹没，同时让每回合的决策更丰富——先清掉那颗碍事的黑子，再落子到关键位置，这个组合拳的感觉会很爽。

### 2.5 抽卡节奏

固定每 3 回合抽一次卡（第 1 回合前先抽一次保证起手有牌）。每次系统生成 3 个成语技能，玩家三选一。第三个选项固定为"自定义"槽位，玩家可输入任意四字成语实时生成。

手牌上限 5 张，每回合最多使用 1 张，用完即弃。

### 2.6 关卡设计

关卡是固定的预生成棋盘布局。每关在标准 15×15 棋盘上预置若干黑子（无五连）。难度通过三个参数控制：

- **黑子数量**：从 6-8 颗（热身）到 100+ 颗（地狱）
- **布局威胁度**：从散落无阵型到多处三连四连
- **AI 水平**：从随机落子到带威胁评估的搜索算法

关卡固定不随机，保证每关经过验证可解。Roguelike 的随机性体现在每次抽到的技能卡不同。

### 2.7 AI 对手

AI 不使用技能，只正常下棋。三档水平：

- **random**：随机选空位。热身关用。
- **basic**：扫描所有空位，对每个空位评估进攻分和防守分，加权取最高。
- **intermediate**：在 basic 基础上加浅层 minimax 搜索。

AI 不需要"理解"技能。技能改变棋盘后，AI 下一步自然基于新的棋盘状态决策。

### 2.8 对手吐槽

AI 对手在关键时机说一句吐槽，增加趣味性。预写文案按类别随机抽取：

- 开局：挑衅/自信
- 被技能影响：按影响程度从不痛不痒到暴怒
- 局势变化：白子成三连/四连时紧张，黑子成四连时得意
- 结局：赢了嘲讽，输了不服

## 三、核心循环

```
进入关卡 → 棋盘上已有预置黑子
  → 抽初始手牌（三选一）
  → 每回合：
      1. 若是抽卡回合（每 3 回合）→ 三选一抽卡
      2. 玩家行动：可选先使用一张手牌，然后必须落一颗白子
      3. 胜负判定（白子五连 → 过关）
      4. AI 行动：落一颗黑子
      5. 胜负判定（黑子五连 → 失败，可重试）
      6. 对手吐槽（视局势触发）
  → 过关 → 进入下一关
```

## 四、技能沙盒

### 4.1 数据结构

```
Board = ("black" | "white" | null)[15][15]
```

整个游戏状态就是这一个二维数组。

### 4.2 API

技能函数签名：`(api: BoardAPI) => void`

**读操作**（5 个）：

| 方法 | 返回值 | 说明 |
|------|--------|------|
| `get(row, col)` | `"black" \| "white" \| null` | 获取指定位置的棋子 |
| `findAll(color)` | `{row, col}[]` | 找出所有指定颜色的棋子 |
| `findEmpty()` | `{row, col}[]` | 找出所有空位 |
| `countInDirection(row, col, color, dr, dc)` | `number` | 从某位置沿方向计数连续同色棋子 |
| `getBoardSize()` | `15` | 获取棋盘尺寸 |

**写操作**（5 个）：

| 方法 | 说明 |
|------|------|
| `place(row, col, color)` | 在空位放子 |
| `remove(row, col)` | 移除棋子 |
| `transform(row, col, newColor)` | 变色 |
| `move(fromRow, fromCol, toRow, toCol)` | 移动到空位 |
| `swap(r1, c1, r2, c2)` | 交换两位置内容 |

**交互操作**（2 个）：

| 方法 | 说明 |
|------|------|
| `requestChoice(candidates)` | 玩家选一个坐标 |
| `requestArea(center, range)` | 玩家选一个区域 |

写操作校验失败时静默跳过。技能执行后引擎扫描：白子五连则直接胜利，黑子五连则回滚技能并退回手牌。

### 4.3 生成约束

- 单次技能最多影响 5 个格子
- 不能直接让白子连成五子
- 不能移除全部黑子
